Git Sync

Use the following endpoints to sync git repositories:

  • GET /pull - Pull changes from a git repository
  • POST /push - Push changes to a git repository

The following query parameters are supported:

  • branch=<branch> - The branch to pull or push
  • repository=<repository> - The repository to pull or push. Only http/https are supported
  • since=<duration> - When pulling, only return changes since the given duration. Example: since=1h30m
  • after=<timestamp> - When pulling, only return changes after the given timestamp (RFC3339). Example: after=2025-02-13T08:00:00Z

Pull returns the following headers

  • X-Git-Head, with the Commit ID of the head
  • X-Git-IsPartial, boolean whether the bundle is partial (or full)
  • X-Git-Hash, a hash of the commit ID and options used (since and after)
The filename of the bundle is also set to git_<commit ID>_<hash>.bundle

Authentication

To authenticate to the 'repository', set the Authorization header to 'Bearer <token>' where token is the token for the repository

Metrics

Metrics are available at /metrics
